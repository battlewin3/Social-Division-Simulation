### 2.1 严重问题（影响复现可靠性）

#### 问题1：【关键】输出目录未自动创建

**位置**: `01_run.R` 第25-31行, `02_results.R` 第23-31行, `03_results_redistribution.R` 第23-31行

**描述**: 代码定义了 `outputdir` 和 `filesdir` 两个目录路径，但从未检查它们是否存在或自动创建。如果 `output/` 和 `files/` 目录不存在，`save.image()` 和 `ggsave()` 操作将报错。

**当前代码**:
```r
outputdir <- file.path(homedir, "output")
filesdir <- file.path(homedir, "files")
# 无 dir.create() 调用
```

**修复方案**:
```r
outputdir <- file.path(homedir, "output")
filesdir <- file.path(homedir, "files")
# 确保目录存在
if (!dir.exists(outputdir)) dir.create(outputdir, recursive = TRUE)
if (!dir.exists(filesdir)) dir.create(filesdir, recursive = TRUE)
```

**优先级**: 🔴 高 — 首次运行必然失败

---

#### 问题2：【关键】随机种子管理不完整

**位置**: `01_run.R` 第192行

**描述**: 虽然通过 `set.seed(thisrow$seed)` 在每个循环开始时设置种子（第192行），但：
- 全局级别未设置种子
- 如果中途中断并重启，无法恢复之前的状态
- 种子仅用于区分模拟运行，未记录代理人生成的种子流

**当前代码**:
```r
set.seed(thisrow$seed)  # 在循环体内部
```

**修复方案**: 在 `lapply` 循环之前添加全局种子设置，并考虑使用 `withr::with_seed()` 或 `set.seed(i * 1000 + thisrow$seed)` 确保嵌套随机过程的可复现性。

```r
# 在 loopdf 生成之前
set.seed(2024)  # 主种子，确保全局复现性

# 在循环内部
set.seed(thisrow$seed * 10000 + i)  # 确保每个 (seed, i) 组合唯一
```

**优先级**: 🔴 高 — 影响完全复现保证

#### 问题4：【中等】`plyr`与`dplyr`冲突风险

**位置**: `01_run.R` 第9-10行, 第12行

**描述**: 同时加载 `plyr` 和 `dplyr` 可能导致函数名冲突（例如 `summarise`、`mutate`、`rename` 等）。加载顺序影响最终命名空间。`plyr` 已被作者 Hadley Wickham 宣布退休（retired）。

**当前代码**:
```r
require(plyr)    # 已退休

```

**修复方案**:

# 方案A：完全移除 plyr
# 将 rbind.fill() 替换为 dplyr::bind_rows()
# require(dplyr)  # 仅加载 dplyr

# 方案B：明确加载顺序并优先使用 dplyr
require(dplyr)   # 先加载 dplyr（更现代的包）
require(plyr)    # 后加载 plyr（仅用于 rbind.fill）

整个代码库中 `plyr` 的实际使用仅限于 `rbind.fill()` 函数（处理不同列数的data.frame合并）。可以用 `dplyr::bind_rows()` 完全替代。

### 2.2 代码质量问题

#### 问题5：【中等】`require()` 未检查返回值

**位置**: `01_run.R` 第8-20行

**描述**: 使用 `require()` 而非 `library()` 意味着包加载失败时仅产生警告而非错误，可能导致后续代码以难以诊断的方式失败。

**修复方案**:
```r
# 方案A：关键包使用 library()
library(stringr)
library(dplyr)
library(data.table)
library(igraph)

# 方案B：封装为安装-加载函数
ensure_package <- function(pkg) {
    if (!require(pkg, character.only = TRUE)) {
        install.packages(pkg)
        library(pkg, character.only = TRUE)
    }
}
```

**优先级**: 🟢 低 — 虽然不理想，但通常不影响运行（但可能给新手带来困难）

#### 问题6：【中等】变量作用域污染

**位置**: `01_run.R` 第176-641行

**描述**: 主模拟循环中使用 `lapply(loopdf$i, function(i) {...})` 但内部频繁引用外部变量（`loopdf`, `N_groups`, `smallw_factor` 等），使得代码难以独立测试和调试。

**修复方案**: 将循环体提取为独立函数，通过参数传递所有必要信息：
```r
run_single_simulation <- function(i, loopdf, ...) {
    # 所有必要变量作为参数传入
}
fulloutput <- lapply(loopdf$i, run_single_simulation, loopdf = loopdf, ...)
```

**优先级**: 🟢 低 — 不影响运行，但影响代码可维护性和可测试性

### 2.3 潜在数值问题

#### 问题9：【中等】分位点切割的边界处理

**位置**: `01_run.R` 第229-235行, 第253-259行等多处

**描述**: 使用 `quantile()` 生成分位切割点时，如果连续变量中存在大量相同值（例如由于标准化），不同分位的切割点可能相同，导致 `cut()` 函数产生意外行为（空区间）。

**修复方案**:
```r
cutpoints <- unique(quantile(agentsdf$income_i, tmpqs))
if (length(cutpoints) < length(tmpqs)) {
    warning("Non-unique quantile cut points detected.")
}
agentsdf$income_i_q <- cut(
    agentsdf$income_i,
    breaks = cutpoints,
    include.lowest = TRUE
) %>% as.numeric - 1
```

**优先级**: 🟡 中 — 在N=1000时极少出现问题，但N更小时可能出现

---

#### 问题10：【低】浮点数精度比较

**位置**: `01_run.R` 第107-112行

**描述**: 使用 `round() == ...` 来筛选满足条件的参数组合：

```r
tmp <- round(apply(loopdf[, disc_vars], 1, sum), 2) == round(race_indirect * 3/10, 2)
```

虽然已通过 `round()` 处理浮点数比较问题，但阈值 `2` 位小数可能在某些极端参数组合下不够。

**修复方案**:
```r
# 使用 near() 或更鲁棒的容差比较
tol <- 1e-8
tmp <- abs(apply(loopdf[, disc_vars], 1, sum) - race_indirect * 3/10) < tol
```

**优先级**: 🟢 低 — 当前实现对该参数设定有效

## 第三部分：优化建议

### 3.1 性能优化

#### 优化1：使用 `data.table` 的原地操作

**当前状态**: 代码混合使用 `data.table`、`dplyr` 和基础R操作，未充分利用 `data.table` 的性能优势。

**优化方案**:
```r
# 当前：使用基础R的 data.frame 操作
agentsdf$income_i <- thisrow$beta_race_income * agentsdf$race_i + rnorm(thisrow$N_agents, sd = thisrow$luck)

# 优化：使用 data.table 的 := 原地赋值（避免复制）
agentsdf[, income_i := thisrow$beta_race_income * race_i + rnorm(.N, sd = thisrow$luck)]
```

**预期收益**: 对N=1000的规模提升有限（~10-20%），但对更大的N（如探索N=10000）提升显著。

---

#### 优化2：推断阶段并行化

**当前状态**: 推断阶段的 `lapply(1:(N_agents+1), function(j) {...})` 是串行的（第482行），对N+1=1001个代理人的每个都运行回归模型。

**优化方案**:
```r
# 使用 parallel 包
library(parallel)
ncores <- detectCores() - 1
inferencesdf <- mclapply(1:(N_agents + 1), function(j) {
    ...
}, mc.cores = ncores) %>% rbind.fill
```

**预期收益**: 推断阶段占据运行时的绝大部分（为每个代理人运行多个回归）。8核并行可提供约6-7倍加速。

**注意事项**: 
- Windows系统上 `mclapply` 不可用，需要使用 `parLapply` + `makeCluster`
- 需要在集群节点间传递 `agentsdf` 和 `networks` 数据

---

#### 优化3：减少冗余计算

**当前状态**: 回归模型 `lm()` 多次拟合相同的数据（normal model和race model独立拟合）。

**优化方案**:
```r
# 一次性计算所有需要的统计量
m_normal <- lm(earnings_f ~ income_i + ability_i + race_i, data = mydf)
m_race <- update(m_normal, . ~ race_i)  # 使用update避免重新构建model frame
```

**预期收益**: 小幅度提升（~5-10%），因为lm本身已经优化。

### 5.1 短期扩展（1-2周工作量）

1. **多代模型**：让当前代理人的收入成为下一代的"继承性阶层位置"，代际传递形成反馈循环；
2. **信息不完全**：引入代理人对他人特征的观察噪声（测量误差）；
3. **异质推断模型**：不同代理人使用不同的推断策略（如启发式vs回归模型）。

### 5.2 中期扩展（1-3个月）

1. **制度变化冲击**：模拟政策干预（如学校去隔离化、居住整合项目）的效果；
2. **网络动态演化**：允许代理人在不同生命阶段改变其社交网络（如搬家、转学）；
3. **多维度不平等**：添加性别、移民身份等其他社会分层维度。

### 5.3 长期扩展（3-6个月）

1. **实证校准**：使用真实数据（如Add Health、NLSY、PSID）校准模型参数；
2. **反事实政策模拟**：系统性地比较不同整合政策（学校边界重划、住房券、平权行动等）对不平等认知和再分配支持的长期效果；
3. **与其他模型的耦合**：将认知推断模块嵌入更大规模的宏观经济ABM中。
